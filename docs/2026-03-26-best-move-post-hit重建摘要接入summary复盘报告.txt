一、本轮推进目标
按照上一轮《post-hit guidance profile delta 输出复盘报告》的第一条下一步建议，这一轮不再继续扩展 move 级 debug，而是把 best move 的 post-hit rebuild 轻量摘要直接接入 `decision_summary`。

上一轮之后，虽然 move 级已经可以看到：

- `post_hit_guidance_debug`
- `rebuilt_delta_from_base`
- `blended_delta_from_base`

但如果只看 `decision_summary`，
仍然很难快速回答：

- best move 的 post-hit rebuild 到底有没有发生
- 用了多少条 signal
- 增强了多少个 confirmed slot
- 最终 guidance multiplier 相对 base 提升了多少

所以这一轮的目标是：

- 把 best move 的 rebuild 摘要直接放进 `decision_summary`
- 让外部在不展开完整 debug 对象的情况下，也能先读到最关键的四个摘要量

- 正常局面有这些字段
- 无候选局面没有这些字段

这样可以保持：
- `decision_summary` 结构稳定
- 外部消费逻辑不需要分支判断

三、本轮收益
这一轮最大的收益，是把 post-hit rebuild 的核心信息从“需要展开 best move 的完整 debug 对象才能读到”，推进成了“看 `decision_summary` 就能先拿到一份轻量摘要”。

现在只看 summary，就能先回答：

1. best move 的 post-hit rebuild 是否发生
2. 这次 rebuild 重建出了多少条 acting signal
3. guidance debug matrix 里补回了多少个 confirmed slot
4. blended guidance multiplier 相对 base 净增加了多少

这让你在批量看实验结果、日志或回归输出时，
可以先用 summary 快速筛，再决定是否需要展开完整 debug。
