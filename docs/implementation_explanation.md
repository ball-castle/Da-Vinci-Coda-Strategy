# 达芬奇密码策略引擎：模型目的与当前实现说明

## 1. 这套模型要解决什么问题

这个项目的目标，不是做一个只会输出“猜哪张牌”的脚本，而是逐步构建一个用于《达芬奇密码》的**隐藏信息策略引擎**。

当前模型想回答的核心问题有 4 个：

1. 如何维护一张尽可能可信的全场暗牌概率图。
2. 在这张概率图上，应该攻击谁、哪一张、猜什么值。
3. 猜对以后，当前回合是否值得继续压。
4. 如何把对手的行为历史作为证据，反向影响当前推理和决策。

如果用工程化语言表述，这个模型的目的就是：

**先维护 belief state，再在 belief state 上做回合收益决策。**

---

## 2. 当前模型的总体结构

当前后端的主流程位于 `backend/app/core/engine.py`，核心上可以拆成三层：

### 2.1 状态与公共约束层

这一层负责表达“场上到底发生了什么”，核心对象包括：

- `CardSlot`
- `PlayerState`
- `GuessAction`
- `GameState`

它解决的是：

- 多玩家、多槽位的统一状态表示
- 已知明牌、暗牌、惩罚翻牌的公共信息接入
- 猜对 / 猜错对目标槽位的硬约束坍缩

这一层的目标，不是决策，而是保证系统对局面的表达是稳定且可扩展的。

### 2.2 概率推理与行为证据层

这一层负责从合法状态空间中推导每个隐藏槽位的后验分布。

当前系统会同时使用两类证据：

- **硬证据**
  - 排序规则
  - 颜色约束
  - 猜对 / 猜错导致的 slot 级固定与排除
  - 惩罚翻牌进入公共状态

- **软证据**
  - 对手猜测值与其自身手牌结构的一致性
  - 猜测值与目标槽位区间的一致性
  - 猜中后是否继续回合
  - 命中后场面的可攻击性

这部分对应 `DaVinciInferenceEngine` 与 `BehavioralLikelihoodModel`。

### 2.3 决策层

这一层不再只问“最高概率是什么”，而是综合评估：

- 命中收益
- 猜错风险
- 信息增益
- 命中后的 continuation value
- 当前应继续还是停手

这部分由 `DaVinciDecisionEngine` 负责。

当前的决策逻辑已经从单步 EV，推进到了：

- stop / continue 显式比较
- post-hit 浅层 rollout
- post-hit top-2 gap 稳定性判断
- post-hit top-k follow-up 支撑聚合

也就是说，系统现在已经不只是看“这一手值不值”，而是开始看“这一手命中以后，这一回合后面还能不能稳定赚”。

---

## 3. 当前模型已经做到什么程度

截至目前，这个模型已经具备以下能力：

1. 能基于结构化状态对象运行，而不是只依赖散字段输入。
2. 能对全场多个玩家的隐藏槽位做联合概率推理。
3. 能把动作日志作为硬约束和软证据同时接入后验更新。
4. 能输出：
   - `best_move`
   - `top_moves`
   - `probability_matrix`
   - `full_probability_matrix`
   - `decision_summary`
   - `should_stop`
5. 能把“继续猜 / 停止”从单步阈值判断，推进到浅层 rollout 驱动的 stop/continue 比较。

换句话说，当前模型已经不再是 demo 级的“猜牌器”，而是进入了：

**有状态骨架、有后验分布、有行为证据、有回合内决策逻辑的策略引擎雏形。**

---

## 4. 当前模型还不是什么

为了避免误判项目成熟度，也需要明确当前模型还不是什么：

1. 它还不是完整的 POMDP 最优求解器。
2. 它还不是完整的多步博弈树搜索系统。
3. 它还没有完成真正的 `P(action | state)` 动作似然模型。
4. 它还没有把“摸黑牌还是摸白牌”正式接入主决策流程。
5. 它还没有完整的性能基线、固定 benchmark 和更系统的回归评测体系。

因此，当前最准确的定位不是“最终成品”，而是：

**一个已经进入阶段 1 中后段，并正在向更强决策层推进的策略引擎。**

---

## 5. 当前模型的直接用途

当前这套模型最适合的用途是：

1. 用作算法迭代平台，持续验证新的 belief / decision 设计。
2. 用作《达芬奇密码》局面分析器，查看暗牌概率分布与推荐动作。
3. 用作后续更重搜索算法的基础层，而不是直接跳到 MCTS 或 Expectimax。

它当前最重要的价值，不是“绝对战胜所有人类玩家”，而是：

**把《达芬奇密码》的推理问题，从经验驱动推进到可解释、可验证、可迭代的策略建模。**

---

## 6. 下一阶段最合理的方向

基于当前结构，下一阶段最合理的推进方向仍然是：

1. 建立固定 case 集与回归测试体系。
2. 把行为层继续从 heuristic 推进成更完整的动作似然模型。
3. 在保证 belief 层可信的前提下，再继续补强更深一点的 continuation rollout。

一句话总结：

**这套模型的目的，是为《达芬奇密码》建立一个先做隐藏信息推理、再做整回合收益决策的策略引擎，而当前工程已经完成了这个方向上的核心骨架。**
