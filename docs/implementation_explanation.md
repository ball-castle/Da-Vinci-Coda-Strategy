# 达芬奇密码极客推演终端 - 当前实现说明与整体框架

基于《达芬奇密码》不完全信息博弈的特殊性，本项目已经成功搭建了一个前后端分离的全栈应用，将复杂的贝叶斯概率推理与现代化的 React 前端界面完美结合。由于本项目拥有 Auto 自动执行授权，所有算法、接口和 UI 层面的整合已经自动实现。以下是关于各个部分以及整体框架的详细说明：

## 一、 整体系统框架 (System Architecture)

该系统主要分为两层：**React 可视化前端（决策控制台）** 与 **FastAPI 算法后端（AI 大脑引擎）**。

1. **前端 (Frontend)**：由 Vite + React 构建，负责提供用户友好的“玻璃拟态 (Glassmorphism)” 暗黑风格交互界面。它将玩家输入的“我的手牌”、“场上明牌（敌我）”以及“敌方猜测历史”统筹为 JSON，发送至后端的智适应算法引擎。
2. **后端 (Backend)**：由 Python FastAPI 驱动，是整个项目的灵魂所在。它负责接收场面状况，并在后台重构出数百、上千个平行宇宙的卡牌树，最后输出理论上“最佳”的一击。

### 核心运作流
环境数据输入 (Frontend) ➔ `POST /api/turn` (API) ➔ 状态空间搜索 DFS (Backend) ➔ 贝叶斯心理战权重过滤 ➔ 期望收益 EV 评估 ➔ 结果渲染 (Frontend)

---

## 二、 后端核心模块说明 (Backend Engine Breakdown)

后端代码位于 `backend/app/core/engine.py`，它深刻践行了项目文档（`VinCi.txt`）里提出的 5 大底层目标，被抽象化为三大模块：

### 1. 推理引擎 (DaVinciInferenceEngine) - "物理排雷"
这个模块是 AI 系统的**左脑基础**。
它利用**深度优先搜索（DFS）**和**约束满足规则（Constraint Satisfaction）**，遍历剩余的未知卡牌池。
* **规则剪枝**：如果一种随机组合不满足“数字递增”或同数字“黑左白右”的硬性规则，分支会被当场剪断。
* **Joker（百搭牌）兼容**：在这个版本中对 `-`（Joker）进行了严密的逻辑兼容：它能够放置在任意位置而不触发递增警报。

### 2. 心理战与概率过滤器 (PsychologicalFilter) - "贝叶斯读心术"
这个模块是 AI 系统的**左脑升华**，解决了人类玩家最难量化的“动作反推”博弈（根据对手猜什么，反推他手里没什么）。
* **负向防自杀原则 (Penalty Self-Absence)**：对手猜过的牌，通常他自己手里是没有的。如果 DFS 算出的某套手牌里刚好包含对手“猜过”的牌，这套组合的可能性权重将被砍至 `0.1` 倍（降低 90%）。
* **正向锚点效应 (Boost Anchor)**：对手极其喜欢通过自己手里的牌作为“锚点夹角”去试探别人。如果组合里包含了对手猜过数字的相邻数字，这会被视为合乎常理的逻辑推演，给予 `1.2` 倍的正向权重奖励。

### 3. 决策引擎 (DaVinciDecisionEngine) - "期望收益"
这个模块充当 **理性的右脑**。它不再做非黑即白的抉择，而是把过滤完的概率矩阵抛入一道**数学期望（Expected Value, EV）**计算题：
* **动态风险系数**：公式中的惩罚不仅取决于命中率，还取决于你当前拥有的**暗牌数量**。你藏在暗处的牌越少，你猜错被迫公开的风险就呈指数级拉大，系统会变得极其保守。
* **终极输出（Best Move）**：遍历整个敌方的暗牌位置，计算每一张的 EV，最终向前端返回一个包含“目标索引”、“极大概率猜测数字”、“该牌胜率”、“总 EV 期望值”的字典。

## 三、 API 接口说明 (Backend Endpoints)

位于 `backend/app/api/endpoints.py`。
* **路由**：`POST /api/turn`
* **职能**：将前端通过 Pydantic 校验的格式（如 `my_cards`, `public_cards`, `opponent_history`）反序列化进 Python Controller，触发核心的 `run_turn` 方法，并将最终得出的 `best_move` 以及遍历的 `search_space_size` 重启发送给前端展示。

---

## 四、 前端交互界面说明 (Frontend React App)

前端位于 `frontend/src/`，摒弃了基础原生的白底黑字，采用极具科技感和黑客氛围的设计，带给使用者视觉上的 Wow 体验。

### 1. 状态配置面板 (State Configuration)
* 用户可以输入当前自身的卡牌阵列 (`my_cards`)。
* 分离输入我方和敌方场上已经翻开的“明牌”。
* **动态历史注入**：可以通过界面输入对手已经尝试过哪些失败的猜测，为后端的心理战模块提供真实的喂料日志。

### 2. 决策控制台 (Decision Engine Output)
当发起“获取最优胜率决策”请求时，系统会在右侧通过发光卡片向用户通报：
* 对手场上应当指认的**绝对位置编号**
* 需要吐出的**颜色与数字**
* 这个行动的**后验胜率**与被验证的**平行宇宙搜索数目**
* 极具特色的是，当**动态 EV 小于 0**时，控制台将变为高风险警告红框，建议玩家见好就收，打出 **STOP 终止执行**以保全底牌。
