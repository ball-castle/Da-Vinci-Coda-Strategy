# 达芬奇密码策略模型复盘与当前进度方向报告

更新时间：2026-03-23

## 1. 文档目的

这份文档用于回答三个问题：

1. 这个项目当前到底已经做到了什么。
2. 这几轮算法优化的核心价值在哪里。
3. 下一阶段应该沿着什么方向继续推进。

这份文档与 `direction/` 目录中的路线文档配套使用：

- `direction/algorithm_direction.txt`
- `direction/algorithm_roadmap.txt`
- `direction/algorithm_remaining_optimizations.txt`


## 2. 项目当前定位

这个项目的目标，不是做一个简单的“猜牌脚本”，而是逐步构建一个用于《达芬奇密码》的策略推理引擎。

当前整体架构仍然遵循最初在 `VinCi.txt` 中确定的主线：

1. 先建立可靠的状态表示与概率推理层。
2. 再在概率层之上做行动决策层。
3. 最后才考虑更重的博弈搜索与性能极限优化。

换句话说，当前项目的核心不是“直接输出猜哪张牌”，而是“系统能否稳定维护一张可信的暗牌概率图”。


## 3. 当前已完成的核心工作

### 3.1 状态模型统一

后端已经不再完全依赖原先零散的：

- `my_cards`
- `public_cards`
- `opponent_history`

而是已经抽出了统一状态层，核心模型包括：

- `CardSlot`
- `PlayerState`
- `GuessAction`
- `GameState`

当前后端已经能够围绕结构化状态对象运行，而不是只靠散字段拼接逻辑。

### 3.2 前后端结构化输入打通

后端接口现在已经支持结构化 `state` 输入。

同时，为了避免一次性打断现有流程，也保留了旧 payload 的兼容转换逻辑。

前端也已经开始改为发送结构化 `state`，这意味着后续继续扩展多玩家、多槽位、动作证据时，不必再继续堆兼容补丁。

### 3.3 从单点猜牌输出升级到概率矩阵输出

系统不再只输出一个简单的 `best_move`。

现在已经能够返回：

- `best_move`
- `top_moves`
- `probability_matrix`
- `full_probability_matrix`
- `should_stop`

这意味着系统已经具备了“先看概率分布，再做决策”的中间层，而不是直接从输入跳到结果。

### 3.4 从单目标推理升级到全场联合推理

最早的版本主要围绕单个 `target_player` 做推理。

当前后端已经进一步推进为：

- 联合枚举多个非自身玩家的隐藏槽位
- 在统一未知牌池下做全场联合分配
- 输出多个玩家的 `full_probability_matrix`

虽然这还不是完全意义上的高阶联合信念网络，但已经从“单目标局部推理”推进到了“全场联合推理骨架”。

### 3.5 心理战权重与动作证据接入

系统已经具备基础的行为加权能力。

当前已经纳入的证据包括：

- 猜过某张牌，则该猜牌者自己持有该牌的组合被惩罚
- 猜测相邻牌时，给予弱正向奖励
- 猜对后继续回合，给予额外行为自信权重

这部分仍然是启发式的、轻量的，但已经开始从“纯规则排雷”往“动作反推”推进。

### 3.6 Slot 级公共信息坍缩

这是本轮非常关键的进展。

当前系统已经能够真正利用：

- `target_player_id`
- `target_slot_index`
- `result`

来做 slot 级的公共信息坍缩：

- 如果某次猜测命中，则目标槽位会直接坍缩为该牌。
- 如果某次猜测失败，则该目标槽位会排除该猜测牌。

这意味着动作日志不再只是“读心权重参考”，而是已经开始成为真正的公共约束。

### 3.7 惩罚牌公开进入公共状态

现在动作日志还支持携带：

- `revealed_player_id`
- `revealed_slot_index`
- `revealed_color`
- `revealed_value`

也就是说，当某次猜错后产生公开惩罚牌时，后端已经能把这张牌直接纳入公共信息。

其效果包括：

- 自动进入 `known_cards`
- 自动从未知牌池中移除
- 自动减少隐藏槽位数量
- 自动缩小搜索空间

这一步非常重要，因为它让“动作导致的状态变化”真正进入了系统，而不是仍然停留在人工解释层。

### 3.8 从固定目标升级到全场自动选目标

当前决策层已经不再只对 `target_player` 评分。

现在系统可以：

- 从 `full_probability_matrix` 中汇总所有玩家的候选行动
- 对所有候选行动做统一评分
- 自动从全场选出最优攻击对象

这意味着 `best_move` 已经具备“跨玩家选目标”的能力，而不是只能对单个目标玩家做固定攻击。


## 4. 这几轮优化的核心价值

这几轮优化最重要的价值，不是简单多写了几个类或字段，而是把系统从“概念上讲得通”，推进到了“工程上开始具备闭环骨架”。

可以概括为三句话：

1. 系统已经有了统一状态模型。
2. 系统已经有了全场概率层。
3. 系统已经有了开始消化公共动作证据的能力。

如果从算法演进角度看，这一阶段真正完成的是：

从“单步猜牌 demo”
推进到
“有统一状态、有全场概率矩阵、有行为证据更新的策略骨架”。


## 5. 当前仍然没有完成的关键部分

虽然进展已经很大，但当前仍然没有进入“强策略引擎完成态”。

以下能力仍未完成，或者只完成了初级版本：

### 5.1 更完整的动作似然模型

当前系统仍然主要使用启发式规则：

- 猜过的牌惩罚
- 相邻牌奖励
- 连续行动加权

但还没有真正建成更完整的“动作似然模型”：

- 为什么他会猜这个玩家？
- 为什么会猜这个槽位？
- 为什么会猜这个值？
- 在某种手牌假设下，这个动作到底有多合理？

### 5.2 更深的公共信息坍缩

当前已经支持：

- 猜对/猜错的槽位级坍缩
- 惩罚牌公开导致的状态更新

但还没有完全做完：

- 公开新牌后对所有玩家候选空间的更深层重压缩
- 多条动作链叠加后的全局重估

### 5.3 继续猜 / 停止的多步评估

当前 `should_stop` 仍然更接近单步 EV 判断。

还没有做到：

- 猜对一次之后重新展开一层未来状态
- 再评估继续猜的长期收益
- 与“本轮见好就收”做多步比较

### 5.4 抽黑还是抽白

这是最初目标中的重要一项，但目前仍未正式接入。

也就是说，系统现在更偏“猜牌与打击决策”，还没有开始处理抽牌颜色选择。

### 5.5 系统化测试集与基准评估

当前仍然缺少：

- 固定测试局面集
- 单元测试
- 基准性能测试
- 决策质量对比集

这意味着现在我们可以判断“功能有没有推进”，但还不能稳定判断“每一次修改是否让系统整体更强”。

### 5.6 前端仍然主要是示例驱动

前端虽然已经切到了结构化 `state`，但当前仍然主要依赖示例数据。

还没有真正做到：

- 可编辑的槽位面板
- 可编辑的动作日志面板
- 多玩家完整局面录入
- 全场概率矩阵可视化

此外，当前前端目录本地还未安装依赖，因此前端这几轮改动尚未完成完整构建验证。


## 6. 当前阶段判断

如果结合 `direction/algorithm_roadmap.txt` 来看，当前项目已经不再是“阶段 1 的早期”。

更准确的判断是：

**当前处于阶段 1 的中后段。**

原因是：

- 状态模型已经开始统一
- 结构化输入已经打通
- 全场联合概率推理已经上线
- 动作证据已经能形成真实公共约束
- 决策层已经能跨目标玩家选最优攻击对象

但之所以还不能说进入阶段 2，是因为：

- 动作似然模型还不够深
- 回合内多步评估还没接起来
- 抽黑/抽白还未进入系统
- 测试和基准体系仍然缺失


## 7. 当前最合理的下一阶段方向

当前最合理的方向，不是马上上 MCTS，也不是马上追求位运算极限优化。

当前最值得继续推进的是：

### 方向 A：补强动作似然模型

优先级很高。

也就是从现在的“启发式行为权重”，继续推进到更强的：

- 猜谁
- 猜哪张
- 猜几
- 为什么继续猜

都参与后验计算。

这是当前从“会读一点心”走向“真的理解行为”的关键一步。

### 方向 B：补强公共信息坍缩的全局传播

虽然惩罚牌已经进入状态，但后续还可以继续增强：

- 新公共牌公开后，对所有玩家的候选空间同时重压缩
- 多条动作证据叠加后，重新组织全局概率空间

### 方向 C：建立测试局面集和验证体系

这是工程上非常关键的一步。

如果没有测试体系，后续每次优化都很难判断到底是“更强了”还是“只变复杂了”。

建议尽快建立：

- 固定局面集
- 预期候选分布
- 关键行动结果
- 搜索空间与响应时间记录


## 8. 当前不建议抢跑的方向

以下方向并不是不重要，而是不应该在当前阶段抢跑：

- MCTS
- Expectimax
- Theory of Mind 完整版
- 位运算 / LUT / bitboard 级性能极限优化

原因很简单：

如果当前的概率层和验证层还不稳定，那么越复杂的搜索，只会建立在更不稳的基础上。


## 9. 结论

截至 2026-03-23，这个项目已经从“构思阶段”推进到了“有统一状态骨架和全场联合推理能力的可持续开发阶段”。

当前最值得肯定的进展有：

1. 状态模型已经统一。
2. 前后端结构化状态链路已经打通。
3. 全场联合概率矩阵已经出现。
4. Slot 级动作证据已经能触发公共信息坍缩。
5. 惩罚牌公开已经真正进入后验更新。
6. 决策层已经能全场自动选目标。

当前最需要继续补的部分有：

1. 更完整的动作似然模型。
2. 更强的全局公共信息传播。
3. 多步继续猜 / 停止评估。
4. 抽黑还是抽白。
5. 固定测试集与验证体系。

如果要用一句话总结当前最合理的推进策略，那就是：

**先把概率推理层和验证层做扎实，再进入更强的决策层与更重的博弈搜索。**
