2026-03-27 联合动作模型全场传播搜索与量化基准补齐复盘报告

一、这次补的不是局部点，而是四个主缺口

这次改动对应上一份评分表里明确列出来的 4 个缺口：

1. `P(action | state)` 还不是更完整的联合动作模型
2. 公共信息坍缩还没有形成更硬的全场联合传播
3. 当前仍然主要是 heuristic + rollout，搜索层不够明确
4. benchmark 还缺少“强多少”的量化

所以这次没有继续补 explanation，也没有继续拆局部 heuristics，
而是直接把这四点做成一批主干更新。


二、这次的核心改动

1. 补联合动作模型

在 `BehavioralLikelihoodModel` 里新增了 `joint_action_fit_breakdown`。

这一步的意义不是多了一个 bonus，
而是把原来分散的：

- target player selection
- target slot selection
- value selection
- continue decision

第一次显式合成了同一个动作联合核。

现在一个动作不再只是“这几个局部项各自乘一下”，
而是会再额外判断：

- 这些局部选择是不是互相一致
- 当前 target / slot / value / continue 是否在同一条行动计划上
- 当前局面的攻击窗口、坍缩传播、bridge、chain 是否支撑这条计划

这让行为似然更接近“联合动作选择”，
而不是原来那种拆开的局部权重拼接。


2. 补全场传播

新增了 `global_public_propagation_pressure`。

以前的 `joint collapse` 已经有用了，
但更多还是“近期 reveal / failed guess 的局部压缩感”。

这次推进之后，
系统开始把下面这些量统一汇总成全场传播信号：

- 多个玩家最近的公开 reveal
- 多个玩家最近的 failed guess
- 各玩家当前 public exposure
- 各玩家当前 finish fragility
- 坍缩是否在多目标之间形成 breadth

然后这个传播信号被接进了：

- player attackability
- continue joint collapse
- single move value
- continuation value
- final continue boundary

也就是说，
公共事件现在不再只是一跳 bonus，
而是已经能跨层回灌进整套主决策。


3. 补显式搜索层

这次没有直接上完整 MCTS，
但已经把原先偏隐式的 post-hit rollout，
推进成了一个更明确的 shallow branch search。

新增的 branch search 会在 post-hit rollout 里：

- 取 top branches
- 同时看 immediate edge 和 future edge
- 用 win probability / continuation likelihood / support ratio 做分支权重
- 汇总出 `branch_search_value / margin / support_ratio / signal`

然后这个结果不只是 debug 输出，
而是正式进入：

- post-hit continuation value
- continue boundary support
- decision summary

这意味着当前系统已经不只是“看 best move 和 top-k 平均值”，
而是开始显式评估“下一层分支整体有没有继续空间”。


4. 补 benchmark 量化

新增了：

- `benchmark_decision_cases`
- `benchmark_behavior_cases`

这一步的价值是把原来的 regression case，
从“通过/不通过”推进成“有量化强度”的固定 benchmark。

当前输出的量包括：

- accuracy
- average continue margin
- average stop margin
- margin separation
- min correct margin
- average log margin
- average score ratio

这批之后，
我们终于不只是能说“没回退”，
还能说“当前这批固定样本上分离度是多少”。


三、这次对五个目标的影响

目标 1 摸黑还是摸白
- 这次不是主攻 draw
- 但因为全场传播和 branch search 被补强，draw 后续接这些共享信号会更顺
- 属于间接增强，不是本批最大受益项

目标 2 猜谁猜哪张猜几
- 这是本批直接增强项
- 因为联合动作核把 target / slot / value 真正统一了

目标 3 猜对后继续还是停
- 这是本批另一个直接增强项
- 因为 continue 已经开始显式读取 branch search 和 global propagation

目标 4 通过别人行为反推他的牌
- 提升也很明显
- 因为行为似然不再只是局部 fit，而是开始判断一整条动作计划是否合理

目标 5 综合成完整算法
- 这次是很实的推进
- 因为四条主缺口第一次在同一批里一起往前走


四、这次完成后的状态

完成这批之后，项目已经从：

- `局部 attackability`
- `局部 collapse`
- `局部 rollout`

进一步推进到了：

- `联合动作一致性`
- `全场传播信号`
- `显式分支搜索`
- `可量化 benchmark`

这意味着模型现在更像一个“共享信号驱动的统一策略核”，
而不是很多局部优秀 heuristic 的叠加。

六、这批最重要的价值判断

如果说之前几批做的是：

- 攻击窗口
- 失败恢复
- 收尾链
- 连压历史

那么这次做的是把这些分散信号进一步统一起来。

最关键的变化不是某个数值变大了，
而是系统第一次开始回答下面这个问题：

“一个动作为什么在这个局面下是合理的整套动作计划，而不只是一个局部看起来不错的猜法？”

这正是从强 heuristic 往更完整策略模型走的关键台阶。


七、还没有完全结束的地方

这次虽然把四个缺口都推进了一大步，
但还没有到“彻底补完”的程度。

还剩的硬点主要是：

1. 联合动作模型现在仍然是结构化加权核，不是真正的全概率生成模型
2. 搜索层现在是 shallow branch search，还不是更完整的 expectimax / MCTS
3. benchmark 现在已经量化了，但还主要基于固定 regression case，不是更大规模对局评测
4. draw / guess / continue 的统一共享核已经更近了，但还没有彻底并成一个总策略目标


八、一句话结论

这次不是“多补了几个字段”，
而是把项目从“很多强局部策略”推进到“开始具备统一动作模型、全场传播、显式分支搜索和量化评测”的阶段。
