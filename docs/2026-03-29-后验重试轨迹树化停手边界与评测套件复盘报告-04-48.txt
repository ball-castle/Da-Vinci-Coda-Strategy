一、本轮目标
- 继续压缩 `P(action | state)` 在弱 case 上的后验分离度
- 让 continue boundary 进一步吃到 tree/search 信号，减少局部 threshold 残留
- 把长期评测从单个 synthetic / 小矩阵，推进到更正式的 evaluation suite 入口

二、本轮核心改动
1. 后验重试轨迹改造
- 新增 `retry_after_failure_trajectory_prior`，把“同槽位失败后按相邻数继续试探”从原来的统一惩罚，改成按 `step / slot_attackability / attack_window` 计算的轨迹先验。
- 这直接修复了 `same_slot_retry` 这类行为 case 的最弱后验分离问题。

2. continue boundary 树化加深
- `stop_threshold_breakdown` 新增 `search_depth_credit` 和 `guard_relief_signal`。
- 当 `strategy_objective_core + tree_search_signal + mcts depth/node count` 足够强时，会主动削弱 `low_confidence_threshold / weak_continuation_threshold`。
- `low_confidence_guard / weak_edge_guard / self_exposure_guard` 的边界也开始读取这个 tree-driven relief，而不是只靠局部硬阈值。

3. 长期评测入口正规化
- 新增 `benchmark_long_horizon_evaluation_suite`，默认走更大的 multi-seed / multi-config / repeated matrix 聚合。
- 评测结果额外输出 `fairness_gap` 和 `balance_score`，不再只看裸 `seat_bias`。
