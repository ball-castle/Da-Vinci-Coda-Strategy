3/24 复盘报告

项目：Da Vinci Code Strategy
日期：2026-03-24

一、本轮工作目标
本轮工作的核心目标，不是做普通代码优化，而是把项目往“更接近最优策略”的方向推进。具体聚焦三件事：
1. 提高隐藏信息推理质量，提升 belief state 的可信度。
2. 提高行动决策质量，让系统不再只做单步高命中率贪心。
3. 在不推翻现有架构的前提下，改善核心推理效率和后续可扩展性。

二、当前项目阶段判断
根据已读取的仓库内容，这个项目当前处于：
“规则约束驱动的隐藏信息推理系统，正在向 belief-state + decision optimization 的策略引擎演进”的阶段。

当前主链路大致是：
GameState -> DFS 枚举合法隐藏状态 -> 叠加行为/心理启发式 -> 生成 probability matrix -> 计算 EV -> 排序并输出 best_move / should_stop

这说明项目已经跨过了“纯规则模拟”阶段，但还没有真正进入“高可信 posterior + 中长期收益决策”的阶段。

三、本轮识别出的核心问题
本轮重点识别出 3 个最影响策略最优性的核心问题。

1. 硬约束与软启发式混在同一 posterior 中
当前 probability matrix 并不是纯 hard inference 的结果，而是被行为 heuristic 混入后的混合分布。
问题在于：
- 规则信息与行为信息没有拆层；
- 概率矩阵缺少可解释性；
- 后续决策层实际上是在一个被污染的 posterior 上做动作排序。

这会直接影响：
- 概率可信度；
- 决策质量；
- 后续升级到更强动作似然模型的可扩展性。

2. DFS 搜索仍然偏“晚剪枝”
当前搜索虽然已经有顺序约束，但很多公共信息并没有尽早进入候选域，而是等到较后阶段才体现。
主要表现为：
- failed guess 没有充分前置成 forbidden domain；
- hit / reveal 没有充分前置成 fixed slot；
- 缺少更强的槽位级最小域优先展开；
- 缺少更系统的 dead-state cache / 前向检查。

这会同时拉低：
- 推理质量；
- 搜索效率；
- 多玩家局面下的稳定性。

3. 决策层仍然过于单步贪心
当前决策更接近：
“选当前命中概率 / 单步 EV 更高的动作”
而不是：
“最大化本回合甚至更长 horizon 的收益”

主要问题在于：
- should_stop 仍然较粗；
- continuation value 没有真正进入主决策；
- 信息增益和收益评估还偏局部。

这会导致：
- 有些该继续压的局面不敢压；
- 有些该保守停手的局面仍可能误冲；
- 行动排序与真实长期胜率之间仍有偏差。

四、本轮给出的主要改造方向
本轮建议的方向不是推翻重写，而是在当前架构基础上做“高收益、最小可落地”的增强。

1. 推理层拆分为五段式流水线
建议把核心流程明确拆为：
1) state representation
2) hard inference
3) soft weighting
4) probability normalization / posterior blending
5) decision scoring

这样做的价值是：
- belief state 更清晰；
- 概率矩阵重新成为一等公民；
- 行为模型后续可升级为真正的 likelihood model；
- 决策层可以明确知道自己在基于什么 posterior 决策。

2. 引入 HardConstraintCompiler
把历史命中 / 失败 / 公开信息编译成：
- fixed_by_slot
- forbidden_by_slot

这样可以让公共信息在 DFS 前置生效，而不是叶子阶段再处理。

3. 将 PsychologicalFilter 独立为 BehavioralLikelihoodModel
不再把 heuristic 当作“约束”，而是作为 posterior 上的软似然重加权。
这一步的意义很大：
- hard posterior 保持纯净；
- heuristic 的影响强度可控；
- 后续能自然演进为 P(action | hidden_state)。

4. 将 DFS 升级为 MRV + forward checking + dead-state cache
搜索顺序从“按玩家顺序展开”改为“按槽位最小候选域优先展开”，并配合前向检查和死状态缓存。
收益主要在于：
- 更早剪枝；
- 更少重复搜索；
- 更适合多玩家局面；
- 性能提升与推理质量提升是同步的。

5. 决策层引入 continuation-aware EV
把当前动作价值从：
EV = 命中奖励 - 猜错代价 + 粗糙信息增益
推进到：
EV = immediate_value + continuation_value

也就是显式估计：
“这手如果猜中，下一手继续猜值不值”

这一步虽然还不是完整博弈树，但已经比纯单步贪心更接近真实策略目标。

五、本轮产出物
本轮已产出以下可落地内容：
1. 一版完整可替换的 engine.py
   - 包含 hard / soft 解耦
   - 包含 MRV DFS
   - 包含 continuation-aware decision
2. 一组基础测试文件
   - 推理正确性测试
   - 决策层基础测试
3. 一套阶段性总结与 commit message 草案

六、本轮改动预期收益
1. 对隐藏信息推理质量
- 概率世界更干净；
- 失败猜测与命中信息进入更早；
- 候选域收缩更充分；
- posterior 更可信。

2. 对决策质量
- 不再只看当前这一下；
- 开始考虑“猜中后还能赚多少”；
- should_stop 更接近真实回合 EV 判断。

3. 对运行效率
- 晚死分支减少；
- 重复搜索减少；
- 多玩家局面下复杂度更可控。

4. 对长期演进能力
- 后续升级动作似然模型更自然；
- 后续做更深 rollout 更容易；
- 后续做更强缓存与位运算优化更有基础。

七、本轮仍然没有解决的问题
需要明确，这一轮还没有解决以下更深层问题：

1. 行为模型仍然主要是 heuristic
虽然已建议把它从 hard inference 中拆出，但它还不是严格的 P(action | state)。

2. continuation 仍然只是轻量近似
目前最合理的是先做 1-step 或浅层 rollout，而不是直接上大型搜索树。

3. 状态表示还没有做到极致高效
例如：
- bitmask 压缩状态
- canonical state hash
- 更强子树 memo
这些仍属于下一阶段优化。

八、下一阶段建议
下一阶段最值得继续推进的顺序如下：

第一优先级：
把行为建模升级成真正的动作似然模型，即：
P(action_history | hidden_state)

重点建模：
- 为什么猜这个玩家
- 为什么猜这个槽位
- 为什么猜这个值
- 为什么继续 / 停止

第二优先级：
继续强化公共信息全局传播，而不是停留在局部缩域。

第三优先级：
做 2-step continuation / shallow tree，而不是立刻上 MCTS 或 Expectimax。

九、当前不建议优先做的事
现阶段不建议优先投入的方向：
- UI / 前端表现优化
- 大规模重构非核心接口层
- 过早引入复杂搜索树
- 纯粹命名/格式层面的低收益修改

原因很明确：
belief 不稳，搜索越深越容易把错误放大。

十、结论
本轮工作的核心成果，不是“让 AI 命中率稍微高一点”，而是把整个项目向真正的策略系统方向推了一步。

一句话总结：
这轮改造的价值，在于把项目从“带 heuristic 的猜牌程序”，推进成“具有 belief-state 架构雏形的隐藏信息决策引擎”。

后续只要继续沿着：
hard inference -> action likelihood -> continuation decision
这条主线推进，项目就会越来越接近“最终输出接近最优策略”的目标。
