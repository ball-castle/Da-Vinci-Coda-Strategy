一、本轮目标

这轮继续按 VinCi.txt 的主线收口，不补解释，直接打 4 个主缺口：

1. 让 `P(action | state)` 的工程化后验更像“生成式动作分布”，把 candidate ranking 直接接到行为后验上
2. 继续抬高 behavior posterior 的区分度，减少个别 case 的低 margin
3. 把 post-draw opening 的首猜 top1 质量真正打穿，解决 Joker 轻微压过数值牌的问题
4. 让 long-horizon benchmark 的多配置聚合更系统，不再被小配置等权平均放大小样本噪声

二、本轮核心推进

1. 联合动作后验又往前推进了一层“后验自校准”
- 在 `joint_action_generative_probability_breakdown` 里补了基于 posterior marginals 的二次整形：
  - `posterior_player_mass`
  - `posterior_slot_mass`
  - `posterior_value_mass`
  - `posterior_continue_mass`
- 现在不是只把 raw posterior softmax 归一化后直接拿来用，
  而是让 player / slot / value / continue 这 4 条边缘后验重新反哺联合动作概率。
- 这一步让 `joint_action_posterior` 更接近“条件联合动作生成近似”，也直接把 behavior benchmark 的区分度往上抬了一截。

2. 候选动作级别的行为后验正式接进 opening 排序
- 新增了 `_behavior_action_posterior_summary`，它会对当前 candidate move：
  - 重建单条 `GuessSignal`
  - 调 `behavior_model.explain_signal(...)`
  - 读取 `joint_action_posterior`
  - 输出 `probability / weight / observed_rank / support`
- 这意味着 opening 期不再只看当前 slot 分布本身，
  还会显式读“这个动作在行为模型的联合动作后验里到底站不站得住”。

3. opening precision 这轮真正把 Joker 压回去了
- `_opening_precision_breakdown` 这轮新增并强化了：
  - `posterior_signal`
  - `joker_penalty`
  - `numeric_competition_signal`
- 关键逻辑变化是：
  - 如果 Joker 只是略高于最佳数值牌，或者根本没有明显拉开，
    就触发更强的 `numeric_competition_signal`
  - Joker 的 `posterior_signal` 会被额外衰减
  - 最终 `support` 再乘上更强的 `OPENING_JOKER_PENALTY_SCALE`
- 同时 opening 排序加入了 `opening_joker_penalty` 作为直接 tie-break，
  所以现在不是“Joker 只要概率略高就天然占优”，
  而是“只有 Joker 真正明显领先时才允许排第一”。

4. 多配置 benchmark 这轮补了“按实际局数加权”
- `benchmark_long_horizon_configuration_matrix` 之前虽然已经有多 seed / 多配置，
  但 `seat_bias` 和 `average_starting_player_advantage` 还是按配置等权平均，
  这会让小配置的随机波动被放大。
- 这轮改成按 `total_game_count` 加权：
  - `seat_bias`
  - `average_starting_player_advantage`
  - `average_match_seat_bias`
- 这一步没有让 benchmark 变快，但让配置矩阵的解释方式更接近“真实长期评测”。

5. 测试层补了 opening 对 Joker 的硬回归
- `test_opening_precision_support_prefers_clearer_post_draw_guess` 现在不仅校验：
  - sharper > blurrier
  - Joker 有 penalty
- 还新增了近似真实 benchmark 的“Joker 和数值牌近乎打平”场景，
  断言 near-tied Joker 不能再压过 near-tied numeric。
- 再加上现有的 long-horizon configuration matrix regression，这轮不是只改排序而没有防回退。

五、仍然没彻底打穿的点

1. `P(action | state)` 还是工程化后验近似，不是完整可学习的概率生成模型
2. 显式 MCTS 仍然是局部后续动作树，不是完整全局 game-state tree / 完整 MCTS
3. `min_log_margin = 0.099684523007778` 仍然不算很硬，说明个别行为 case 的 posterior separation 还有提升空间
4. synthetic self-play 的 opening benchmark 已经很强，但它本身仍偏 pre-draw / post-draw 首猜视角，不能单独代表完整长期进攻能力
5. 多 seed 标准 matrix 已经基本压平先手优势，但更大规模、多配置、可长期复跑的 benchmark 还没有真正做大
6. `stop / draw / risk guard` 仍然残留局部 threshold 和 shaping，尚未被统一目标完全吞掉

这轮不是补解释，而是把“opening 首猜精度”和“benchmark 聚合质量”两个最后阶段最影响体感强度的问题直接往前推了一大步。
