达芬奇密码策略模型剩余优化清单

这份清单用于回答一个非常实际的问题：

现在算法优化还缺什么？
哪些是必须马上做的？
哪些可以放到下一阶段？
哪些应该放到后期？


一、当前结论

当前项目已经完成了第一阶段的一部分地基工作：
- 已经有统一的状态模型
- 已经有统一的动作模型
- 已经有结构化 state 输入
- 已经有基础概率矩阵输出
- 已经有基础心理权重

但这还不等于“算法优化完成”。

目前更准确的判断是：

我们已经完成了“从散乱逻辑走向统一骨架”这一步，
但还没有完成“从统一骨架走向强推理引擎”这一步。


二、必须马上做的优化

这部分属于当前阶段最关键的内容，不做的话，后面的高级策略和搜索都容易建立在不稳定基础上。

1. 全场联合状态推理
- 现在主要还是围绕一个 target_player 在算
- 还不是“全场所有玩家、所有暗位”的联合推理
- 这是当前阶段最应该补齐的核心能力

2. 公共信息坍缩
- 某次猜错后翻出一张新牌
- 某张牌从暗牌变成明牌
- 某个公共边界被收紧
- 这些变化都应该立刻反向压缩全场状态空间

3. 动作证据充分利用
- 现在动作结构已经有了
- 但推理层还没有充分利用：
  - 猜的是谁
  - 猜的是第几张
  - 猜对还是猜错
  - 猜对后有没有继续
- 这些证据还没有完整进入后验更新

4. 固定测试局面集
- 如果没有标准 case，我们就不知道每次修改是变强了还是变弱了
- 当前必须建立一组固定局面，作为算法对照集

5. 基础验证体系
- 至少要有：
  - 合法性测试
  - 概率分布测试
  - 状态转换测试
  - 动作日志解析测试


三、下一阶段应该做的优化

这部分适合在第一阶段后半段和第二阶段开始时推进。

1. 继续猜 / 停止的回合内评估
- 当前 should_stop 更接近单步 EV
- 还不是真正的“继续猜一轮之后的长期收益评估”
- 下一步应做轻量 rollout 或回合内递推

2. 抽黑还是抽白
- 这是目标 1
- 目前还没有实装
- 需要基于概率矩阵和手牌缝隙共同计算

3. 更完整的动作似然模型
- 当前心理战还是启发式权重
- 后面要逐步升级成更接近贝叶斯动作似然
- 也就是：
  - 某种底牌组合下
  - 对方做出这个猜测动作的合理性有多高

4. 可解释结果输出
- 当前已经能输出 top_moves 和概率矩阵
- 但后面最好还能告诉前端：
  - 为什么推荐这一步
  - 为什么不建议继续猜
  - 哪些证据影响了这次后验结果


四、后期再做的优化

这部分不是不重要，而是不应该在当前阶段抢跑。

1. MCTS / Expectimax / 深层 rollout
- 这属于强策略阶段
- 在概率层不稳定之前上这些会浪费很多精力

2. Theory of Mind / 换位思考
- 包括公共缝隙分析
- 包括从对手视角重算他为什么会这么猜
- 这会非常强，但也更复杂

3. 性能极限优化
- 位运算
- 预计算表
- 更轻量的数据结构
- 联合状态缓存
- 这些适合在模型正确性基本稳定后再推进


五、按类别拆开的剩余问题

功能层还缺的：
- 全场联合推理
- 公共信息坍缩
- 抽黑/抽白
- 多步继续猜/停止
- 更强的换位思考

推理层还缺的：
- 动作似然模型
- 多玩家联动更新
- 更完整的 joint belief
- 位置证据的充分使用

性能层还缺的：
- 缓存
- 预计算约束
- 更轻量的数据结构
- benchmark 体系

工程验证层还缺的：
- 固定测试局面集
- 单元测试
- 前端可编辑状态面板
- 完整前端构建验证


六、优先级总表

P0：必须马上做
- 全场联合状态推理
- 公共信息坍缩
- 动作证据充分利用
- 固定测试局面集
- 基础验证体系

P1：下一阶段做
- 继续猜 / 停止 rollout
- 抽黑还是抽白
- 更完整的动作似然模型
- 更强解释输出

P2：后期做
- MCTS / Expectimax
- Theory of Mind
- 位运算 / LUT / 高性能缓存


七、一句话总结

现在最缺的，不是“更复杂的搜索算法”，而是“更完整、更可验证的概率推理能力”。

所以当前最正确的推进方向依然是：

先补强全场状态推理和验证体系，
再补强决策层，
最后再考虑更重的博弈搜索和性能极限优化。
